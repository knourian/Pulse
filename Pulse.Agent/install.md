

# Pulse Agent Installation Guide

This guide explains how to install and run `Pulse.Agent` on:

* Windows
* Linux (systemd)

The installation uses the deployment files:

* `publish.bat`
* `install-windows-service.bat`
* `install-linux-service.sh`
* `pulse-agent.service`

---

# 1. Prerequisites

## .NET Runtime

The target machine must have the .NET 10 runtime installed.

Verify:

```bash
# Windows
    dotnet --list-runtimes

# Linux
    dotnet --list-runtimes
```

Ensure a .NET 10 runtime is installed before running the agent.

If not installed:

* Install the correct .NET runtime version used by the project.

---

# 2. Publish the Agent

Run:

```bat
publish.bat
```

This generates:

```text
publish/
├── win-x64/
└── linux-x64/
```

---


# 3. Edit Configuration

Before installation, edit:

```text
appsettings.json
.env
```

Important values:

```json
{
  "Server": {
    "BaseUrl": "https://your-server-url"
  }
}
```

Replace:

```text
https://your-server-url
```

with your actual Pulse server URL.

Example:

```text
https://pulse.company.local
```


Rename Environment File

Rename:
```text
.env.example
```
to:
```text
.env
```
Update Encryption Key
Inside .env update:

```text
Agent__IdentityEncryptionKey=replace-with-a-long-random-secret-at-least-16-chars
```

Replace it with a strong random secret.

Example:


```text
Agent__IdentityEncryptionKey=9c4e2a7f1b8d6e3c5a1f7d9e2b4c6a8f

```

Requirements:

- minimum 16 characters
- should be random
- should be unique per environment
- should not be committed to git

---

# 4. Windows Installation

## 4.1 Copy Files

Copy these files to the target machine:

```text
publish/win-x64/
install-windows-service.bat
```

---

## 4.2 Install Service

Run as Administrator:

```bat
install-windows-service.bat
```

The installer will:

* create installation directory
* copy files
* create Windows service
* start service automatically

---

## 4.3 Verify Service

Open:

```text
services.msc
```

Verify:

```text
PulseAgent
```

is:

```text
Running
```

---

## 4.4 Windows Service Commands

### Start

```powershell
sc start PulseAgent
```

### Stop

```powershell
sc stop PulseAgent
```

### Delete

```powershell
sc delete PulseAgent
```

---

# 5. Linux Installation

## 5.1 Copy Files

Copy these files to the target machine:

```text
publish/linux-x64/
install-linux-service.sh
pulse-agent.service
```

---

## 5.2 Make Installer Executable

```bash
chmod +x install-linux-service.sh
```

---

## 5.3 Install Service

Run:

```bash
sudo ./install-linux-service.sh
```

The installer will:

* create `/opt/pulse-agent`
* copy files
* install systemd service
* enable service
* start service automatically

---

## 5.4 Linux Service Commands

### Status

```bash
sudo systemctl status pulse-agent
```

---

### Start

```bash
sudo systemctl start pulse-agent
```

---

### Stop

```bash
sudo systemctl stop pulse-agent
```

---

### Restart

```bash
sudo systemctl restart pulse-agent
```

---

## 5.5 Linux Logs

View logs:

```bash
journalctl -u pulse-agent -f
```

---

```text
C:\Pulse.Agent
```

---

## Linux


# 7. Files Generated by Agent

The agent may generate runtime files such as:

```text
identity.json
logs/
```

These files should remain writable by the service user.

---

# 8. Updating the Agent

## Windows

1. Stop service
2. Replace files in installation directory
3. Start service

Commands:

```powershell
sc stop PulseAgent
sc start PulseAgent
```

---

## Linux

1. Stop service
2. Replace files in `/opt/pulse-agent`
3. Restart service

Commands:

```bash
sudo systemctl stop pulse-agent
sudo systemctl restart pulse-agent
```

---

# 9. Troubleshooting

## Agent Does Not Register

Verify:

* server URL is correct
* server reachable from machine
* firewall allows access
* Pulse.Server is running

---

## Linux Service Fails

Check:

```bash
journalctl -u pulse-agent -n 100
```

---

## Windows Service Fails

Check:

* Windows Event Viewer
* application logs
